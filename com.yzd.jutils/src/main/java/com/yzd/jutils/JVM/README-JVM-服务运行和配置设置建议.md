## 服务运行和配置设置建议
```
jvm信息
连接池
timeout设置
1-jvm信息:
当前的jvm的参数一般都是dockerfile控制的，一般只需要修改xmx xmn xms三个参数，具体的参考需要自己评估一下内存的使用情况，以及监控的信息来决定 
如果出现fgc的解决方法，如果确实是内存使用比较大，就提高最大jvm内存数，如果内存使用不大，通过jmap来dump内存数据，然后mat分析泄漏点
2-连接池:
连接池的设置一般的原则是根据两点预估qps以及io操作的平均响应时间来共同决定，比如qps是1000，redis的操作时10ms，那么理论上一个连接1s可以100个请求，所以预估10条连接可以hold住请求，可以再上浮20%作为冗余
连接池不易设置过大，需要根据存储的承载情况来决定连接数的上线，比如mysql的最大不能大于5000左右吧，redis一样。
需要搜集充分的需求，设计合理的连接池的大小
连接池的监控 通过JMX监控druid和jedis的连接池的状态
3-timeout设置:
timeout设计的比较多，也是需要根据具体的情况来设计吧，比如项目一的有些超时设置的非常大，有的是120s，有的是300s。再比如门店那边有些就是5s。这个还是需要根据产品的性能要求，qps等等各方面的线索，做一个合理的设置
```
